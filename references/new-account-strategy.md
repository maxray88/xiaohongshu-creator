# New Xiaohongshu Account Strategy — 零风控起号

## Phase 1: 账号隔离 (Day 1)

### 设备/网络要求
- **全新设备或干净环境**：同一台 Mac 上的 Chrome 新 profile（非默认 user-data-dir）
- **独立 IP**：不要用家庭 WiFi（已关联旧账号），使用手机热点或公司网络
- **独立手机号**：需要一个新的手机号注册小红书

### Chrome 配置
```bash
# 创建全新 profile（不与旧账号共享任何 cookie/storage）
# Chrome → 偏好设置 → 用户 → 添加新用户
# 命名如 "XHS-New" — 这样 user-data-dir 自动分离
```

### 注册流程（纯手动）
1. 用新手机号注册（不要在脚本里自动填信息）
2. 完善基本信息：头像（真实照片风格）、昵称（不营销化）、简介（生活化）
3. **关键**：头像和简介要用真实感的，不要用 AI 生成的

## Phase 2: 养号期 (Day 2-14)

### 行为模式（纯手动，绝对不要脚本介入）

| 天数 | 行为 | 频率 | 时长 |
|------|------|------|------|
| Day 2-3 | 正常浏览，搜索感兴趣的关键词 | 2-3次/天 | 10-15分钟/次 |
| Day 4-7 | 开始点赞、收藏、偶尔评论 | 3-5次/天 | 15-20分钟/次 |
| Day 8-14 | 正常用户行为混合 | 5-7次/天 | 20-30分钟/次 |

### 禁止行为（养号期）
- ❌ 任何自动化操作（点赞、评论、发布、关注）
- ❌ 大量关注/取关
- ❌ 短时间内集中操作（如 10 分钟点赞 50 次）
- ❌ 使用任何脚本/插件

### 推荐行为
- ✅ 浏览不同类别的内容（生活、美食、旅行、学习）
- ✅ 偶尔在感兴趣的帖子下写**非模板化**评论
- ✅ 关注 10-20 个真实创作者（不同领域）
- ✅ 收藏一些帖子（不批量）

## Phase 3: 首篇内容 (Day 15)

### 发布策略
- **首篇必须是原创个人内容**，不是营销/干货类
- 建议：生活日常、个人感悟、一次旅行/美食体验
- 图片：真实拍摄，非网图
- 文字：口语化，允许不完美（有 typo 都没关系）

### 脚本介入时机
- 首篇**建议手动发布**，完全建立"真人"行为模式
- 脚本从**第 2-3 篇**开始介入（使用带 warm-up 的新版 xhs_publish.py）

## Phase 4: 脚本介入 (Day 16+)

### 使用新版脚本
```bash
# 默认存草稿（自动热身行为：浏览+点赞+评论+回家）
python3 xhs_publish.py --title "标题" --content "正文" --images "封面.jpg"

# 如需发布（审核后）
python3 xhs_publish.py --title "标题" --content "正文" --images "封面.jpg" --publish
```

### 行为模式
- 脚本每次运行会**自动热身 3-8 分钟**（随机时长）
- 浏览 → 点赞 → 评论 → 回首页 → 发布
- Cron 时间已**随机化**（10:00-20:00 之间随机），避免固定时间触发

## 关键区别：旧账号 vs 新账号

| 维度 | 旧账号（情绪树洞） | 新账号 |
|------|---------------------|--------|
| 风控评分 | 高（历史自动化标记） | 零（全新） |
| 内容检测 | 严格（连存草稿都触发） | 宽松（正常内容可通过） |
| 行为检测 | 严格 | 宽松 |
| 推荐策略 | 长期静养 + 行为伪装 | 先养号 14 天再发布 |

## 注意事项

1. **新账号也需要静养**：注册后至少 2-3 天不发布任何内容
2. **IP 一致性**：同一账号使用同一 IP，频繁切换 IP 会触发风控
3. **内容指纹**：避免批量生成高度相似的文案
4. **设备指纹**：不要在同一设备/浏览器上频繁切换多个账号
